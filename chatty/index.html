Add chatyman in two steps

Step 1 — Add the script

Host chatyman.js on your site and add one tag before </body>:

<script src="/chatyman.js" defer></script>

Step 2 — Author /chatyman.json

Describe a tiny flow and the pages your bot may read:

{
    "branding": { "name": "Helpdesk" },
    "flow": {
        "start": "root",
        "nodes": {
            "root": {
                "say": "Hi! How can I help?",
                "buttons": [
                    { "label": "Pricing", "goto": "pricing" }
                ],
                "fallback": "site_rag"
            },
            "pricing": {
                "answer": {
                    "text": "Our plans start at $0.",
                    "sources": ["/pricing/"]
                }
            }
        }
    },
    "site": {
        "include": ["/", "/pricing/*"],
        "maxPages": 40
    }
}

Frequently asked questions

Does it need a special browser?

No. chatyman works in every modern browser as a smart scripted bot, with embeddings running on WASM everywhere. Generative answers need WebGPU; where it's available, the in-browser LLM layer switches on automatically.

Is my data safe?

Yes. Conversations run in-browser and same-origin — nothing is sent to our servers or any third-party LLM provider. There's no per-message server cost because the work happens on the visitor's device.

Can it answer questions about my site?

Yes. chatyman performs scoped retrieval over your own pages and answers with citations. It only reads the pages you allow — never the open web, and it won't invent facts.

How is it cheaper?

There's no per-chat server cost: the visitor's browser does the work. That means unlimited conversations with no metered per-conversation fees.
